Configuration

Configuration (map, options)

lowlight.astar.js (line 867)

This library works with reusable configurations. Once data is initialized, just call the path method to compute a path. If you need to partially update your graphs, it is possible to do it manually. If your graphs need to be totally updated, it may be better to just create a new configuration.

Use options.worker to relieve main thread from heavy computations and be able to perform others tasks during calculations. This options uses Web Workers. Note however, that it comes with several constraints (such as the configuration cannot be edited directly, etc.). In most cases, the default usage (i.e. not using options.worker should be enough).

  • Arguments

    map Object Map
    options Object See below
    heuristic
    Default : Heuristics.manhattan     		String Heuristic function. Accepted values are defined in Heuristic object
    heuristicOptions
    Default : {}     		Object Default options for heuristic options
    cost
    Default : function     		Object Cost function. Parameters are nodes' data
    diagonals
    Default : false     		Boolean Enable or disable diagonals movement
    cutting
    Default : false     		Boolean If enabled, diagonals movements between two blocking cells are allowed
    torus
    Default : false     		Boolean If enabled, map will be considered as a torus (map is wrapped on itself)
    order
    Default : "yx"     		String Order of array access value (in arrays structure). Accepted values are "xy" and "yx"
    worker
    Default : ""     		String If enabled, methods will be computed in a separate thread. Pass file's source as parameter to enable

  • Simple usage

    //Define some variables
let map = [], X = 30, Y = 30

//Define a biome function (optional)
let biome = (x, y) => {
   let e = noise(x, y) //Elevation
   if (e <= 0.1) return -1.0 //Sea
   if (e <= 0.2) return  1.1 //Beach
   if (e <= 0.4) return  1.0 //Plains
   if (e <= 0.5) return  1.2 //Forest
   if (e <= 0.7) return  1.4 //Hills
   if (e <= 0.9) return  1.6 //Mountains
   if (e <= 1.0) return  2.0 //Snow mountains
}

//Create map
for (let x = 0; x < X; x++) { map[x] = []
   for (let y = 0; y < Y; y++) {
       map[x][y] = biome(x, y)
   }
}

//Define configurations options
let options = {
    order:"xy",
    diagonals:true,
    cutting:false,
    torus:false,
    heuristic:Lowlight.Astar.Heuristics.euclidian
}

//Create a new configuration
//You can define multiples layers with differents parameters and/or cost functions while using the same map
let astar = new Lowlight.Astar.Configuration(map,
   {...options, cost() { return m >= 0 ? m : NaN }}, //Only terrestrial fields accepted
   {...options, cost() { return m < 0 ? -m : NaN }}, //Only aquatic fields accepted
   {...options, cost() { return Math.abs(m) }} //All fields types accepted
)

//Compute a path
astar.path({x:0, y:0}, {x:0, y:0}, {
   callback(path, scores) { path.map(n => console.log(n)) },
   layer:0 //Terrestrial
})

path (start, goal, options) → {Promise.<Array.<Vertex>>}

lowlight.astar.js (line 974)

Compute a path from start to end. The heuristic used by this method must be stored in Heuristic. Values in scores are the sum of total cost from start to current node and heuristic cost from current node to goal. Scores aren't available when options.worker is enabled.

You must use an admissible heuristic function to find the shortest past.
When options.jps is enabled, movements cost should be uniform, as JPS makes assumptions on them to prevent nodes expansions.

  • Arguments

    start Object Start vertex
    goal Object Goal vertex
    options Object Options
    layer
    Default : 0     		Number Layer to use (if multiple graphs)
    heuristic
    Default : Heuristics.manhattan     		function Heuristic function to use
    jps
    Default : false     		Boolean Execute A* JPS (Jump Point Search) to speed up computation (only work on uniform cost graphs)
    fail
    Default : false     		Boolean Throw an error if no path was found
    thread
    Default : false     		Boolean Use worker instead of main thread
